Git Cheat Sheet

Setup Git Username and Password

Initializing a Repository

git init : Create the .git folder

Viewing Status and Logs

git status

git log

git log --online

Staging and Committing Changes

git add . : Stage all files

git add <filename1> <filename2> : Stage only the mentioned files

git rm <filename> : Remove files from working directory and staging area

git commit -m "commit message"

git commit : Editor will open to enter the commit message

git commit --amend : Redo the previous commit message (only the message)

git commit -am "commit message" : Stage and commit changes (**new files won't be affected)

Branching

git branch : List all branches

git branch -v : List all branches with the last commit

git branch <branchname> : Create a new branch

git switch <branchname> : Switch to a branch

git switch - : Switch to the previous branch

git checkout <branchname> : Switch to a branch

git switch -c <branchname> : Create and switch to a branch

git branch -d <branchname> : Delete a branch

git branch -m <branchname> : Rename/move branch (**have to be inside the branch to execute this)

git merge <branch2> : Merge branch2 into the current branch

Viewing Changes

git diff : View only unstaged changes

git diff HEAD <filename> : View all changes (staged and unstaged)

git diff --staged : View only staged changes

Stashing

git stash : Save the work before navigating to another branch

git stash pop : Restore and remove stashed work, changes are removed from the stash once applied

git stash apply : Apply stashed changes without removing them from the stash

git stash -u : Apply stash for unstaged files as well

git stash list : View stash list

git stash apply stash@{0} : Apply specific stashed changes 

**_ git stash pop removes stashes from the list, so
_** we have to use drop only if we use apply

git stash drop stash@{1}

git stash clear : Remove all saved stashes

Detached Head State

git checkout <commithash> : Switch to detached head state for experimentation, then create a new branch and commit new changes to that branch

git checkout HEAD~1 : Point to the previous commit

git checkout HEAD~8 : Point to the previous 8 commits

git checkout HEAD <filename> : Discard changes

git checkout -- <filename> : Same as above

git restore <filename> : Same as above

Resetting and Reverting

git reset <commithash> : Remove the commit, but keep the changes

git reset --hard <commithash> : Remove the commit and the changes

git revert <commithash> : Make a new commit to undo the changes
